Protocol & SSE vocabulary

The wire format between client and backend is the Vercel AI SDK UI Message Stream v1. Reusing it gets us useChat, DefaultChatTransport, and AI Elements components for free. Subagents and a few of our own data channels ride alongside as typed data-<key> events.

Requirements

  1. A backend implementer can write a complete adapter from agent_harness events to our wire format using only this page and its child.
  2. A frontend developer can write a custom renderer for a new tool without reading any backend code.

SSE envelope

  • Transport. HTTP/1.1 chunked or HTTP/2. Content-Type: text/event-stream. Required header x-vercel-ai-ui-message-stream: v1 — without it, @ai-sdk/react falls back to plain-text parsing.
  • Frame. Each event is one data: line followed by a blank line. Payload is JSON.
  • Termination. Final frame is the literal data: [DONE]\n\n.
  • Keepalive. A : ping comment line every ~30s prevents intermediary proxies from idle-closing the connection. The AI SDK client ignores comment lines.

Content-Type: text/event-stream; charset=utf-8

Event vocabulary (what we honor)

The full AI SDK v1 vocabulary is large; we implement a subset that covers everything we actually need. Anything not listed is out of scope until a requirement demands it.

startmessageIdopens an assistant message slot
start-step / finish-step—multi-step agent boundaries (we use this to bracket each LLM call within a turn)
text-start / text-delta / text-endid, deltaassistant markdown, streamed via Streamdown

The mapping is mostly mechanical. The non-trivial pieces are subagents (separate page) and NodeEnter/NodeExit graph events, which we treat as internal and don't surface unless a debugging surface needs them.

Our data-* channels

The AI SDK reserves data-<key> events for custom channels. We use:

KeyPurpose
data-subagent-eventRepublished child events tagged with parent context. See subagent extension.
data-session-titleServer-generated chat title (debounced; one per turn).
data-errorRecoverable mid-stream errors (rate limit, tool failure that's been retried, etc.).
data-keepaliveOptional; carries no payload; lets the client distinguish "stream is alive" from "connection has just been idle."

Each data-* event accepts transient: true to mark it as non-persisted (the AI SDK won't store it on the UIMessage). Subagent events are persisted; keepalive is transient.
