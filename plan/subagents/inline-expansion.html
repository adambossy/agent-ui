One-level-deep inline expansion

When a user clicks the chevron on a subagent's header, the chat surface inlines a mini-transcript that shows the subagent's reasoning, tool calls, and assistant text — using the same renderers as the top-level transcript. One level deep only; further nesting requires a tap-into.

Requirements

  1. An expanded subagent looks and behaves like a small, indented copy of the main chat — same components, same animations, same theming.
  2. Multiple expansions are independent: opening, closing, or scrolling within one does not affect the others.
  3. If a subagent itself invokes a deeper subagent, the deeper one shows as a tappable header inside the expansion, not as a recursive inline render.

The expansion

When expanded, the SubagentTool mounts a SubagentTranscript component inside its content panel. SubagentTranscript takes a subagent sessionId + the messages from the substore for that session and renders them via the same component vocabulary as the main transcript — but inside a contained, indented frame.

// src/components/SubagentTool.tsx (sketch)
export function SubagentTool({ part }: { part: ToolUIPart }) {
  const { sessionId } = part.output ?? {};
  const sub = useSubagentStore(s => s.byToolCallId[part.toolCallId]);
  const [expanded, setExpanded] = useState(false);

  return (
    <Collapsible open={expanded} onOpenChange={setExpanded}>
      <CollapsibleTrigger asChild>
        <SubagentHeader part={part} sub={sub} onTapInto={() => openSession(sessionId)} />
      </CollapsibleTrigger>
      <CollapsibleContent>
        <SubagentTranscript sessionId={sessionId!} messages={sub?.messages ?? []} />
      </CollapsibleContent>
    </Collapsible>
  );
}

The frame is a left-bordered indented panel (~24px of left padding, a subagent-tinted border) so it visually nests without being a separate widget style. Streamdown, Reasoning, Tool, MessageContent — all the same components — render inside it. The transcript reads from the substore, not from the parent's useChat messages array.

Parallel subagents render independently

Each SubagentTool instance subscribes to the substore with a selector keyed by its own toolCallId:

const sub = useSubagentStore(
  useShallow(s => s.byToolCallId[toolCallId])
);

Zustand only re-renders the component whose selected slice changed. The reducer applies each data-subagent-event to exactly one entry in byToolCallId. Result: when subagent A emits a token, only A's SubagentTool repaints; B's stays untouched. Same independence we want for parallel ordinary tool calls.

Visually, parallel subagents stack in their natural arrival order inside the parent's assistant message (since each is a separate tool-<name> part). Each gets its own colored chrome.

Recursion stop rule. A subagent expansion shows nested subagents as tappable headers only. Their inline expansion is disabled in this view — clicking "Open ↗" navigates to that subagent as a first-class session, where its own one-level expansion becomes available. This bounds the DOM and keeps the visual depth legible.

Implementation: SubagentTranscript renders SubagentTool components with a prop allowExpand={false} so the chevron is suppressed. The top-level transcript renders SubagentTool with allowExpand={true}.

Mobile note. The indented expansion frame uses ~24px of left padding on desktop and shrinks to ~12px below the sm breakpoint to preserve content width. The chevron and "Open ↗" affordances remain visible on mobile but tighten into a single tappable row above the transcript. If on real-device testing the expansion still feels cramped, we fall back to "tap-into only" below a width threshold — documented as an active open question in assumptions.
