Branch · Client

Client app skeleton

Vite + React + AI Elements. No Next.js, no Server Components. The client is a pure SPA that talks to the backend over HTTP/SSE. Routing is client-side; the URL is the source of truth for which session is open.

Requirements

  1. A new contributor can clone the repo, run one command, and have the app talking to a mock backend in under a minute.
  2. Deep-linking to any session works on first load — no "session not found" flash.
  3. Switching between sessions is instant if the session is already cached, with a clear loading state otherwise.
  4. Every screen is laid out for phone, tablet, and laptop/desktop from day one; the sidebar collapses to a drawer below the md breakpoint, the composer stays anchored to the bottom on mobile, and nothing requires hover.

Stack

  • Build: Vite 7 + React 19 + TypeScript 5.7.
  • UI: AI Elements (shadcn-style registry, copied into src/components/ai-elements/) + Tailwind v4 in CSS Variables mode + shadcn primitives.
  • Chat hook: @ai-sdk/react's useChat with DefaultChatTransport.
  • Markdown: streamdown + plugins (@streamdown/{code,math,mermaid,cjk}).
  • Routing: react-router (DOM v7). One route: /c/:sessionId?.
  • State: useChat per active session + a Zustand store for cross-session things (open sessions list, session-tree cache). See state & rendering.
  • No Next.js, no Vercel runtime, no Drizzle, no Auth.js. Auth is whatever the backend uses (cookie / bearer).

Routing

PathRenders
/Empty chat surface (new session pending first message)
/c/:sessionIdChat surface bound to that session. Auto-resumes if a stream is in flight.
/c/:sessionId/tree (modal)Session-tree drawer overlay; URL state only — does not unmount the chat behind it.

The chat surface is one component. The sidebar (recent sessions + tree) is another. Both stay mounted; only the session id changes. Switching sessions never unmounts the surface — it swaps the useChat binding.

Responsive layout. Tailwind mobile-first. Below the md breakpoint (768px) the sidebar collapses to a slide-in drawer triggered by a hamburger in the chat header; the chat surface fills the viewport. From md up, the sidebar is fixed in the left rail. The session-tree drawer is the same component on all screens — it's already a modal/sheet by design, so there's nothing to swap. Composer stays bottom-anchored on every viewport; on mobile it uses position: sticky with safe-area insets so it sits above the keyboard. Touch targets are at least 44 × 44 px (Apple HIG) — that's a hard rule for every interactive element.

Transport wiring

// src/chat/transport.ts
import { DefaultChatTransport } from "@ai-sdk/react";
import { backend } from "@/backend";              // ← our interface, see below

export function makeTransport(sessionId: string) {
  return new DefaultChatTransport({
    api: `/api/chat`,                              // POST endpoint
    body: () => ({ sessionId }),                   // appended to every request
    fetch: (input, init) => backend.fetch(input, init),  // pluggable
    // Resume support — points at GET /api/chat/:sessionId/stream
    resumeStreamApi: () => `/api/chat/${sessionId}/stream`,
  });
}

The backend object is our own wrapper around fetch that respects whichever BackendBindings implementation is active (real or mock). In dev it can rewrite URLs to the mock; in prod it's a passthrough with the auth header attached.

App bootstrap

// src/main.tsx
import { createRoot } from "react-dom/client";
import { BrowserRouter, Routes, Route } from "react-router-dom";
import { App } from "./App";
import { ChatScreen } from "./screens/ChatScreen";
import { selectBackend } from "./backend";

selectBackend(import.meta.env.VITE_BACKEND ?? "mock");   // "mock" | "real"

createRoot(document.getElementById("root")!).render(
  <BrowserRouter>
    <Routes>
      <Route element={<App />}>
        <Route path="/" element={<ChatScreen />} />
        <Route path="/c/:sessionId" element={<ChatScreen />} />
      </Route>
    </Routes>
  </BrowserRouter>
);

Two routes, one screen. VITE_BACKEND=mock for solo frontend dev; VITE_BACKEND=real for end-to-end. The App shell holds the sidebar; ChatScreen is the chat surface itself.
