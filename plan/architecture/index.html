Branch · Architecture

Architecture overview

Three actors: the browser running our React app, the backend (an HTTP layer in front of agent_harness), and a stream cache (Redis or equivalent) that keeps mid-flight responses resumable. The client only ever talks to the backend over HTTP/SSE; the backend owns everything else.

Requirements

  1. A developer can run the client locally without a real backend by pointing it at a mock that satisfies the same interface.
  2. A user's tab can disconnect and reconnect mid-stream without losing the answer in progress.
  3. Every concept on the wire — message, tool call, subagent — has a single source of truth on the backend, not the browser.

Data flow for a normal turn

sequenceDiagram participant U as User participant C as Client participant B as Backend participant H as agent_harness participant R as Stream cache U->>C: hit Enter C->>B: POST /chat {sessionId, message} B->>H: agent.start(...) B-->>C: SSE: start, start-step loop streamed H-->>B: MessageDelta, ToolCallDelta, ThinkingDelta... B->>R: tee SSE frame B-->>C: SSE frame (text-delta, reasoning-delta, tool-input-delta, ...) end H-->>B: SubagentStart (if subagent invoked) B-->>C: SSE: data-subagent-event {parentToolCallId, ...} H-->>B: ToolExecEnd / AgentEnd B-->>C: SSE: finish-step, finish, [DONE] C->>U: render assistant message

The harness emits its native Event taxonomy (MessageDelta, ToolCallDelta, SubagentStart, ...). The backend translates each into one or more AI SDK v1 SSE frames. Subagent events live on the same stream tagged with the parent tool-call id — see subagent extension.

The client never reads from any of these directly. It reads from the backend's SSE stream and from a small set of JSON endpoints (GET /sessions/:id, GET /sessions/tree).

Deployment shape (informative)

For local development: vite dev on :5173 proxying /api/* to uvicorn on :8000. For production: static client (any CDN), FastAPI on any Python host, Redis anywhere, an S3-compatible bucket anywhere. No Vercel runtime. No edge functions. The streaming protocol carries no platform-specific assumptions; the marker header (x-vercel-ai-ui-message-stream: v1) is just a content-type tag the AI SDK client reads to choose its parser.
