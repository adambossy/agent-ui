Component inventory

Which AI Elements components we use, which we drop, and where we add our own. AI Elements is shadcn-style: each npx ai-elements add <name> drops a .tsx into our repo that we own and can edit.

Requirements

  1. A frontend developer can swap or restyle any chat component without touching application logic.
  2. The dependency graph stays auditable — every AI Elements file lives in src/components/ai-elements/ and is regular React code.

Keep (install from AI Elements registry)

ComponentRole
Conversation + sub-partsStick-to-bottom container; "jump to bottom" FAB; markdown export.
Message + MessageContent + MessageResponseUser vs assistant bubble; streaming markdown via Streamdown.
MessageActionsPer-turn icon row (copy, regenerate, etc.).
Reasoning + ReasoningTrigger + ReasoningContentAuto-open-on-stream / auto-close 1s after end / "Thought for Ns".
Tool + ToolHeader + ToolInput + ToolOutputDefault tool card — collapsible, status badge, JSON args + output.
PromptInputComposer with attachments + model selector + submit/stop.
AttachmentsInline preview of FileUIParts.
Sources"Used N sources" collapsible footer (if a turn includes source-* parts).
InlineCitationInline numbered references — useful for tools that cite.
CodeBlockShiki + copy + theme. Used by ToolInput/ToolOutput and inside MessageResponse.
Suggestion / SuggestionsFollow-up prompt chips on the empty state.

Drop

  • Artifact + the entire side-panel system. The chat surface is a single column. Generative tool renderers (see generative UI) can still produce rich content inline.
  • Canvas / Node / Edge and the rest of the workflow set. Out of scope.
  • Persona, SpeechInput, AudioPlayer, Transcription, VoiceSelector, MicSelector. No voice in MVP.
  • Sandbox, WebPreview, JsxPreview. Only relevant if we add code execution as a tool; we'll bring them back per-tool when we do.
  • OpenInChat. Aimed at deep-linking to ChatGPT et al.; not useful here.

Add (custom components, owned by us)

ComponentRole
SubagentToolReplaces the default Tool when the tool call is a subagent invocation. Shows agent name + status; expandable to SubagentTranscript; "Open as session" tap-into. See subagent inline expansion.
SubagentTranscriptOne-level-deep mini transcript for an inline subagent expansion. Same renderers as the top-level transcript (Message, Reasoning, Tool) but driven by a child message store.
SessionSidebarLeft rail: recent sessions list + session-tree drawer trigger.
SessionTreeDrawerModal/drawer that renders the full session tree (pi-style); click a node → openSession().
ToolRendererRegistryMaps toolName → custom renderer (see generative UI). Falls back to the default Tool.
UploadDropzoneDrag-drop + file picker wired to the upload interface. Shows progress + retry.

Theming

Tailwind v4 in CSS Variables mode. We inherit the shadcn token set (--background, --foreground, --muted-foreground, --accent, --ring, ...) and define one extra family for subagent chrome (--subagent-border, --subagent-tint). Dark mode flips via the standard .dark class on <html>; auto-detect via prefers-color-scheme with a user override stored in localStorage.
