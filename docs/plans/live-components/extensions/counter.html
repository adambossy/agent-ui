Leaf · Sample extension

counter — the trivial sample

A single integer. Two ops. Two tools. Two buttons in the renderer. Exists to prove the manifest contract isn't over-engineered for simple cases and that two extensions can coexist in one chat.

Requirements

  1. A whole counter extension fits in a single ~50-LOC file.
  2. The user can run a todo-list and a counter in the same chat without either bleeding state into the other.

Schema

const Counter = z.object({
  label: z.string().default(""),
  value: z.number().int(),
  min:   z.number().int().optional(),
  max:   z.number().int().optional(),
});
type Counter = z.infer<typeof Counter>;

Op union

OpPayloadConflict policy
increment{ delta: number }commute-by-delta — concurrent increments sum correctly
set{ value: number }lww — last write wins on a direct set

The commute-by-delta policy is a one-liner in the manifest: when applying an increment with stale baseSeq, the server sums all intervening increments and applies anyway. This is the simplest possible CRDT-flavoured op — and it's why a counter is the right second sample: it exercises the conflict policy machinery without dragging in todo-list complexity.

LLM tools

{
  "counter.increment": z.object({ docId: string, delta: z.number().int() }),
  "counter.set":       z.object({ docId: string, value: z.number().int() }),
}

No create-tool needed; the chat can also pre-seed counters with a system message that calls counter.set({ value: 0 }) at session start.

Renderer

export function CounterView({ doc, dispatch }: LiveRendererProps<Counter, CounterOp>) {
  return (
    <div className="flex items-center gap-3 rounded-md border px-3 py-2">
      {doc.label && <span className="font-medium">{doc.label}</span>}
      <button
        aria-label="decrement"
        onClick={() => dispatch({ kind: "increment", delta: -1 })}
        disabled={doc.min !== undefined && doc.value <= doc.min}
      >−</button>
      <input
        type="number"
        value={doc.value}
        onChange={(e) => dispatch({ kind: "set", value: Number(e.target.value) })}
        className="w-16 text-center"
      />
      <button
        aria-label="increment"
        onClick={() => dispatch({ kind: "increment", delta: 1 })}
        disabled={doc.max !== undefined && doc.value >= doc.max}
      >+</button>
    </div>
  );
}

Demo flow

  1. User: "Start a counter labelled 'pushups today' at 0."
  2. LLM calls counter.set({ value: 0 }); renderer appears with two buttons and a numeric input.
  3. User clicks + 12 times. Twelve op-applied records broadcast.
  4. User: "How many?" LLM reads doc → "Twelve pushups today."
  5. User clicks + at the same instant the LLM calls counter.increment({ delta: 5 }). Both apply because commute-by-delta commutes; final value advances by 6.

This demo is one of two MVP demos. Together with the todo-list demo, they prove the registry dispatches by kind and the contract handles both rich and trivial extensions cleanly.
